# 📋 รายการสิ่งที่ต้องพัฒนาต่อฝั่งหลังบ้าน (Backend Pending Tasks Checklist)

เอกสารฉบับนี้รวบรวม **"รายการที่ต้องทำหลังบ้านทั้งหมด (Backend Backlog)"** ที่ยังไม่เสร็จสิ้นสำหรับการขึ้นระบบจริง (Go-Live) และระบบ Phase 2 ในอนาคต เพื่อให้ทีมพัฒนาสามารถดูและทำงานต่อยอดได้ทันทีโดยไม่ต้องเริ่มต้นนับหนึ่งใหม่

---

## 🗄️ 1. ระบบฐานข้อมูลจริง (Database Integration)

ปัจจุบันหลังบ้านทำงานด้วย In-Memory Mock DB ข้อมูลจะสูญหายเมื่อรีสตาร์ทเซิร์ฟเวอร์

- [ ] **เชื่อมต่อฐานข้อมูล MongoDB จริง**
  - แก้ไขในไฟล์ `data_service.py` สลับจาก mock dictionary ไปรันคำสั่งสืบค้นของ Beanie Document (เช่น `User.find_one(...)` หรือ `Booking.insert(...)`)
  - นำตัวแปร `MONGODB_URL` และ `DATABASE_NAME` ของจริงที่ได้รับจากบริษัทไปอัปเดตใส่ในไฟล์ `.env`
- [ ] **การทำ Database Indexing & Migrations**
  - ตั้งค่า Index ความปลอดภัยในระดับคอลเลกชันฐานข้อมูล (เช่น กำหนดให้ `email` ในคอลเลกชันสมาชิกเป็น Unique Index ห้ามสมัครซ้ำเด็ดขาด)

## 💳 2. ระบบธุรกรรมชำระเงินและคลังสเปซจริง (Payments & Cloud Storage)

- [ ] **การเชื่อมต่อคลังเก็บภาพสลิปจริง (Cloud Image Upload)**
  - ปัจจุบันหลังบ้านมี Endpoint `/api/v1/payments/pay` สำหรับรับอัปโหลดและตรวจความถูกต้องประเภทไฟล์รูปภาพสลิปชำระเงินเรียบร้อยแล้ว
  - สิ่งที่ต้องทำเพิ่ม: เชื่อมต่อ API ของคลังข้อมูลคลาวด์ เช่น **Google Cloud Storage (GCS)**, **AWS S3** หรือ **Supabase Storage** เพื่อโยนไฟล์รูปภาพสลิปที่ได้บันทึกเก็บไว้แบบถาวรแทน URL จำลอง

## 🥈 3. ฟีเจอร์เฟสสอง (Phase 2 - Social Matchmaking & Advanced Features)

หลังบ้านได้ทำโครงสร้าง Endpoint ตัวอย่าง (Mock Router) รอมนสัญญาการต่อท่อ API ไว้หมดแล้ว สิ่งที่ต้องพัฒนาตรรกะระบบเบื้องลึกมีดังนี้:

- [ ] **ระบบล็อกอินด้วย Google OAuth (Social Sign-in)**
  - ปัจจุบัน Endpoint อยู่ที่ `/api/v1/auth/google` (คืนค่าเป็น mock token)
  - สิ่งที่ต้องทำเพิ่ม: นำรหัส `id_token` ที่ฝั่งหน้าบ้านส่งกลับมาไปต่อท่อยืนยันความปลอดภัยกับเซิร์ฟเวอร์ของ Google เพื่อดึงข้อมูลบัญชีจริงมาใช้ลงทะเบียนสมาชิกในระบบ
- [ ] **อัลกอริทึมจับคู่ผู้เล่นอัจฉริยะ (NTRP Matchmaking Algorithm)**
  - ปัจจุบัน Endpoint ค้นหาคู่คือ `/api/v1/matching/find` (ดึงคนที่มีระดับ NTRP ใกล้เคียงกันจาก Mock DB)
  - สิ่งที่ต้องทำเพิ่ม: พัฒนาตัว Algorithm เต็มรูปแบบ คัดกรองตามระดับความต้องการที่ผู้ใช้ระบุ (เช่น ต้องการเจอคนเก่งเท่ากัน/มากกว่า/น้อยกว่า), รูปแบบประเภทการเล่น (เดี่ยว/คู่), สไตล์การเล่น และนัดหมายสร้างแมตช์ลงปฏิทินปฏิทินจองสนามร่วมกัน
- [ ] **ระบบรีวิวผู้ใช้และ UGC Loop**
  - ปัจจุบัน Endpoint อยู่ที่ `/api/v1/matches/{id}/reviews` (บันทึกลง mock reviews)
  - สิ่งที่ต้องทำเพิ่ม: เชื่อมให้รีวิวจริงที่บันทึกเข้าไปทำการคำนวณแบบสะสม ( stars average rating) เพื่อนำไปอัปเดตแสดงผลที่หน้าโปรไฟล์หลักของผู้เล่นจริง
- [ ] **ระบบสแกนและอนุมัติสลิปโอนเงินอัตโนมัติ (Auto Slip OCR)**
  - ปัจจุบันระบบใช้แอดมินเข้าไปตรวจสอบและกดอนุมัติสลิปโอนเงินแบบแมนนวลผ่าน Endpoint ของแอดมิน
  - สิ่งที่ต้องทำเพิ่ม: นำระบบไปต่อท่อ API ตรวจสอบสลิปกับผู้ให้บริการ OCR ในไทย เช่น **Easy Slip API** หรือ **SlipOK API** เพื่อสแกน QR Code บนรูปสลิป ค้นหาข้อมูลยอดเงินและวันเวลาโอนจริงมาเปรียบเทียบกับรายการจอง หากข้อมูลถูกต้องให้ปรับการจองเป็น `confirmed` โดยทันทีโดยไม่ต้องพึ่งแอดมิน
