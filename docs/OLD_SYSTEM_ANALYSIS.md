เอกสารฉบับนี้จัดทำขึ้นเพื่อบันทึก **"ความรู้เชิงลึกของระบบจองเก่า"** ที่ได้รับการถอดรหัสจากเว็บบอร์ดจริง คอลัมน์ไฟล์ข้อมูล เพื่อเตรียมพร้อมสำหรับการแปลงข้อมูล (Data Migration) เข้าสู่ฐานข้อมูล MongoDB จริง

## 🏛️ 1. สถาปัตยกรรมระบบเก่า (Legacy Tech Stack)

จากการเจาะลึกโครงสร้างเว็บไซต์สาธารณะของ [almximpacttennis.com](https://www.almximpacttennis.com/) และระบบจอง [smartbooking.tennissummituniverse.com](https://smartbooking.tennissummituniverse.com/) พบข้อมูลทางสถาปัตยกรรมดังนี้:

*   **Frontend UI:** ใช้ React + Vite + TailwindCSS จัดทำขึ้นเป็นแอปพลิเคชันหน้าเดียว (Single Page Application - SPA)
*   **AI Engine Platform:** พัฒนาขึ้นโดยใช้แพลตฟอร์ม **Lovable.app** (หรือเดิมชื่อ *GPT Engineer*)
*   **Database & Backend Service:** เชื่อมต่อระบบผ่าน **Supabase** (PostgreSQL) เป็นหลัก
*   **ผู้จัดทำโครงการเดิม:** บริษัท **KFU Co., Ltd.**
*   **ที่มาของไฟล์ CSV:** ข้อมูลผู้ใช้ถูกดึง (Export) ออกมาจากฐานข้อมูล Supabase PostgreSQL ตรงๆ ทำให้โครงสร้างข้อมูลเป็นตารางที่มีระเบียบสูง (Relational Schema)

## 📊 2. วิเคราะห์โครงสร้างข้อมูลประชากรผู้ใช้ (User Demographic CSV Analysis)

ตำแหน่งไฟล์ข้อมูล: `database/ALM_Users Demographic.csv` (มีผู้ใช้สมัครเรียน/จองสนาม ทั้งสิ้น **767 คน**)

### 📡 โครงสร้างคอลัมน์และคำอธิบายฟิลด์ข้อมูล (Data Schema):

| ลำดับคอลัมน์ | ชื่อคอลัมน์ภาษาไทย | รูปแบบข้อมูล (Data Type) | คำอธิบายและข้อสังเกตสำคัญสำหรับนักพัฒนา |
| :---: | :--- | :--- | :--- |
| 1 | `ลำดับ` | Integer (1, 2, 3...) | ลำดับการสมัครเฉยๆ |
| 2 | `User ID` | String (UUIDv4) | **รหัสอ้างอิงของสมาชิกหลัก** มาจาก Supabase Auth/Profiles UUID (เช่น `19bdb00f-4655-4fdf-bfc7-87a0971c3480`) ใช้เชื่อมโยงตารางอื่นได้ |
| 3 | `ชื่อ` | String | ชื่อจริง (มีทั้งภาษาไทยและอังกฤษ) |
| 4 | `นามสกุล` | String | นามสกุลจริง (บางแถวปล่อยว่างได้) |
| 5 | `ชื่อเล่น` | String | ชื่อเล่น (สำคัญมากสำหรับวงการเทนนิสเพื่อเรียกจับคู่) |
| 6 | `เบอร์โทร` | String / Integer | **เบอร์มือถือไทยรูปแบบตัดเลขศูนย์ตัวหน้าออก** (เช่น `809132580` = `080-913-2580` / `982815353` = `098-281-5353`) |
| 7 | `อีเมล` | String | อีเมลส่วนตัวที่ใช้ลงทะเบียน |
| 8 | `เพศ` | String (`male`, `female`) | เพศของสมาชิก |
| 9 | `วันเกิด` | Date (`YYYY-MM-DD`) | วันเดือนปีเกิดจริง |
| 10 | `สัญชาติ` | String (เช่น `thai`) | สัญชาติของสมาชิก |
| 11 | `VIP` | String (`Yes`, `No`) | สถานะสมาชิกพิเศษ |
| 12 | `Point สะสม` | Integer | คะแนนสะสมจากการจอง/ร่วมกิจกรรม (มีตั้งแต่ 0 ไปจนถึงหลักล้าน เช่น `1,240,000` พอยต์!) |
| 13 | `สถานะยืนยันตัวตน`| String | สถานะเช็คเบอร์โทร/ตัวตน ได้แก่: `approved` (ยืนยันแล้ว), `pending` (รอตรวจสอบ), `rejected` (ปฏิเสธ) |
| 14 | `วันที่สมัคร` | Date (`D/M/YYYY` พ.ศ.)| **วันที่สมัครในรูปแบบปี พ.ศ. ของไทย!** (เช่น `19/12/2568` = ค.ศ. 2025, `8/4/2569` = ค.ศ. 2026) |

## ⚠️ 3. คำเตือนและสิ่งที่ต้องระวังในขั้นตอนการทำ Data Migration

ในกรณีที่ต้องเขียนสคริปต์เพื่อย้ายผู้ใช้จากไฟล์ CSV นี้เข้าสู่ฐานข้อมูล MongoDB จริงใน Phase 1.5 Go-Live มีจุดวิกฤตที่ต้องระวัง 3 ข้อหลักดังนี้:

1.  **การแปลงรูปแบบเบอร์โทรศัพท์ (Thai Phone Number Format):**
    *   เบอร์โทรใน CSV ถูกเก็บแบบจำนวนเต็ม/ตัดเลขศูนย์นำหน้าออก (เช่น `809132580`)
    *   *สิ่งที่ต้องทำ:* ต้องแปลงเบอร์โดยตรวจสอบความยาว หากมีความยาว 9 หลัก ให้ทำการเชื่อมเลข `0` นำหน้าเสมอ เพื่อให้ได้เบอร์ไทย 10 หลักเต็มรูปแบบ (เช่น `0809132580`) ก่อนจะนำเข้าบันทึก
2.  **การแปลงปี พ.ศ. เป็น ค.ศ. (Thai Buddhist Era Calendar):**
    *   ฟิลด์ `วันที่สมัคร` เก็บข้อมูลปีเป็น พ.ศ. ไทย (เช่น `2568`, `2569`)
    *   *สิ่งที่ต้องทำ:* ต้องทำการดึงส่วนปีของวันที่ออกมา แล้ว **ลบออกด้วย 543** เสมอ เพื่อแปลงให้กลายเป็นปี ค.ศ. สากล (เช่น `2569 - 543 = 2026`) ก่อนเซฟลง MongoDB เป็นฟอร์แมต Datetime ของระบบ
3.  **การสร้างฟิลด์ทดแทนที่ไม่มีในไฟล์ CSV:**
    *   ในไฟล์ CSV ไม่มีข้อมูล `password_hash` (เนื่องจาก Supabase Auth ซ่อนข้อมูลรหัสผ่านที่เข้ารหัสไว้ภายนอก)
    *   *แนวทางแก้ไข:* เมื่อ Import สมาชิกเดิมเข้าไป ให้กำหนดรหัสผ่านตั้งต้น หรือเปิดฟลูการใช้งานการล็อกอินผ่าน OTP เพื่อบังคับให้ผู้ใช้ตั้งรหัสผ่านใหม่ในการเข้าใช้งานครั้งแรก (หรือเชื่อมต่อผ่าน Google OAuth ที่ดึงเมลเดียวกันมาจับคู่)

## 🛠️ 4. ระบบการอิมพอร์ตข้อมูลที่พัฒนาเสร็จสิ้นแล้ว (Completed Migration Script & Seeding)

ระบบได้รับการติดตั้งสคริปต์สำหรับการล้างและโยกย้ายข้อมูลสมาชิก 767 คนโดยอัตโนมัติแล้ว:

### ⚙️ เครื่องมืออิมพอร์ตข้อมูล (`seed_users.py`):
*   **ที่ตั้งไฟล์สคริปต์:** [seed_users.py](file:///c:/GitHub/ALM-X-IMPACT-Tennis/database/seed_users.py)
*   **การรันสคริปต์:** รันผ่าน Virtual Environment ของ Python:
    ```powershell
    backend\venv\Scripts\python database\seed_users.py
    ```
*   **การทำงาน:**
    1. อ่านไฟล์ประชากรผู้ใช้ CSV และจัดเก็บลงตัวแปรทีละแถว
    2. ทำความสะอาดเบอร์โทรศัพท์ (ใส่ศูนย์นำหน้าคืนให้สำหรับเบอร์มือถือ 9 หลัก)
    3. แปลงปี พ.ศ. เป็น ค.ศ. สากลให้อย่างสมบูรณ์
    4. เข้ารหัสความปลอดภัยรหัสผ่านตั้งต้นให้กับทุกคนคือ: **`welcome123`** โดยใช้กลไก Bcrypt จริง
    5. ส่งออกผลลัพธ์เป็นไฟล์เจซอนชื่อ [seeded_users.json](file:///c:/GitHub/ALM-X-IMPACT-Tennis/backend/app/services/seeded_users.json)

### 📂 การโหลดเข้า Memory Database อัตโนมัติ (`mock_db.py`):

    - ข้อมูลส่วนตัว คะแนนสะสมประชากรจริง ปรากฏพร้อมให้ใช้งานทันที!
