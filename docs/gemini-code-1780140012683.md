# สัญญาจ้างพัฒนาระบบและวางระบบอัตโนมัติ

### ข้อ 1. วัตถุประสงค์และขอบเขตของงาน
[cite_start]ผู้ว่าจ้างตกลงจ้าง และผู้รับจ้างตกลงรับจ้างพัฒนาระบบแพลตฟอร์มและระบบอัตโนมัติสำหรับสนามเทนนิส ALM X Impact Tennis โดยมีขอบเขตการทำงานและรายละเอียดการแบ่งระยะโครงการ (Phases) ดังต่อไปนี้: [cite: 12, 13]

#### Phase 1: Foundation & Legacy Integration
[cite_start]**(ระยะเวลาดำเนินการ: 1 สัปดาห์)** [cite: 14, 15]
* [cite_start]ออกแบบและจัดทำหน้าเว็บไซต์หลัก (Landing Page) ที่มุ่งเน้นการเพิ่มอัตราการเปลี่ยนผู้ชมเป็นลูกค้า (Conversion-Focused Design) [cite: 17]
* [cite_start]พัฒนาระบบจองสนามเทนนิสออนไลน์ (Booking System) ที่ใช้งานง่ายและรายงานผลแบบเรียลไทม์ [cite: 18]
* [cite_start]พัฒนาระบบจับคู่ผู้เล่น (Matching System) เพื่ออำนวยความสะดวกในการหาคู่เล่นในสนาม [cite: 19]
* [cite_start]ดำเนินการเชื่อมต่อ บูรณาการ และย้ายฐานข้อมูลจากระบบเก่าทั้งหมด (Legacy Systems) ให้สามารถทำงานร่วมกันได้อย่างสมบูรณ์และไม่มีรอยต่อ [cite: 19]

#### Phase 2: Membership & Advanced Features
[cite_start]**(ระยะเวลาดำเนินการ: 1 สัปดาห์)** [cite: 20, 21]
* [cite_start]พัฒนาระบบจองโค้ชผู้ฝึกสอน (Coach Booking System) พร้อมตารางเวลาการปฏิบัติงานของโค้ชแต่ละท่าน [cite: 22]
* [cite_start]จัดทำระบบการแบ่งระดับสมาชิกและสิทธิพิเศษ (Member Tier & Privilege System) สำหรับสมาชิกระดับต่างๆ [cite: 23]
* [cite_start]พัฒนาระบบกำหนดราคาตามระดับสมาชิก (Member Pricing) เพื่อให้ระบบคำนวณส่วนลดโดยอัตโนมัติตามสถานะสมาชิก [cite: 24]
* [cite_start]สร้างหน้ากระดานแสดงรายงานกิจกรรมของผู้ใช้งาน (User Activity Report) เพื่อการติดตามพฤติกรรมและการวิเคราะห์ข้อมูลการใช้งาน [cite: 26]

#### Phase 3: Marketplace & Service Expansion
[cite_start]**(ระยะเวลาดำเนินการ: 1 สัปดาห์)** [cite: 27, 28]
* [cite_start]พัฒนาระบบร้านค้าออนไลน์สำหรับการขายสินค้า อุปกรณ์ และบริการภายในสนาม (Marketplace) [cite: 29]
* [cite_start]พัฒนาระบบการแลกรับสิทธิ์และคูปองส่วนลด (Voucher Redeem System) [cite: 29]
* [cite_start]พัฒนาระบบ Service Marketplace (Digital Service Layer) เพื่อนำเสนอบริการดิจิทัลและข้อมูลการให้บริการทั้งหมดของสนามอย่างเป็นระบบ [cite: 30]

### ข้อ 2. ระยะเวลาการดำเนินงาน (Project Timeline)
[cite_start]ผู้รับจ้างจะใช้เวลาดำเนินการพัฒนาระบบตามขอบเขตงานในข้อ 1 ทั้งสิ้นเป็นระยะเวลา 3 สัปดาห์ โดยระยะเวลาจะเริ่มนับตั้งแต่วันที่ผู้ว่าจ้างได้ชำระเงินมัดจำงวดแรก (งวดที่ 1) และส่งมอบข้อมูล ภาพถ่าย โลโก้ เอกสาร และข้อมูลหลังบ้านที่จำเป็นสำหรับการพัฒนาระบบให้แก่ผู้รับจ้างครบถ้วนแล้ว [cite: 31, 32]

### ข้อ 3. ค่าจ้างและการชำระเงิน
[cite_start]ผู้ว่าจ้างตกลงชำระค่าจ้างเหมาจ่ายรวมทั้งสิ้นเป็นจำนวนเงิน 100,000 บาท (หนึ่งแสนบาท) (จำนวนนี้ยังไม่รวมภาษีมูลค่าเพิ่ม ถ้ามี) โดยตกลงแบ่งชำระเงินออกเป็น 3 งวด ตามเงื่อนไขความสำเร็จของงานดังต่อไปนี้: [cite: 33, 34, 35]

| งวดที่ | สัดส่วน (%) | เงื่อนไขการชำระเงิน | จำนวนเงิน (บาท) |
| :--- | :---: | :--- | :---: |
| งวดที่ 1 (มัดจำ) | 50% | [cite_start]ชำระทันทีเมื่อลงนามในสัญญาฉบับนี้เพื่อเป็นเงินมัดจำและเริ่มดำเนินงานใน Phase 1 [cite: 36] | [cite_start]50,000 [cite: 36] |
| งวดที่ 2 (งวดสุดท้าย) | 50% | [cite_start]ชำระเมื่อผู้รับจ้างส่งมอบงานใน Phase 2-3 พร้อมทำการทดสอบระบบสมบูรณ์ และส่งมอบงานทั้งหมด [cite: 36] | [cite_start]50,000 [cite: 36] |

### ข้อ 4. การแก้ไขงานและขอบเขตงานที่เพิ่มขึ้น
[cite_start]**4.1** ผู้ว่าจ้างสามารถขอแก้ไขหรือปรับปรุงรายละเอียดงานในแต่ละ Phase ได้ไม่เกิน 2 ครั้งต่อ Phase โดยการแก้ไขนั้นจะต้องไม่กระทบต่อโครงสร้างหลักของระบบ หรืออยู่นอกเหนือขอบเขตงานที่ระบุไว้ในข้อ 1 ของสัญญาฉบับนี้ [cite: 37, 38]

[cite_start]**4.2** ในกรณีที่ผู้ว่าจ้างประสงค์จะแก้ไขงานเกินกว่าจำนวนครั้งที่กำหนด หรือต้องการเพิ่มคุณสมบัติใหม่ที่อยู่นอกเหนือจากขอบเขตงานเดิม ทั้งสองฝ่ายตกลงตีกรอบเป็นงานเพิ่ม โดยผู้รับจ้างจะทำการประเมินราคาและระยะเวลาที่ต้องใช้เพิ่มเติม และทำเป็นบันทึกข้อตกลงแนบท้ายสัญญาก่อนเริ่มดำเนินการในส่วนที่เพิ่มขึ้นนั้น [cite: 39]

### ข้อ 5. การส่งมอบและการรับประกันผลงาน
[cite_start]**5.1** ผู้รับจ้างจะดำเนินการติดตั้งระบบ ส่งมอบ Source Code ตลอดจนสิทธิ์การเข้าถึงและใช้งานระบบให้แก่ผู้ว่าจ้างอย่างสมบูรณ์ เมื่อผู้ว่าจ้างได้ทำการตรวจรับงานและชำระเงินงวดสุดท้าย (งวดที่ 2) ครบถ้วนแล้วเท่านั้น [cite: 40, 41]

[cite_start]**5.2** ผู้รับจ้างตกลงรับประกันความผิดพลาดของระบบ (Bug Fixes) อันเนื่องมาจากความผิดพลาดของชุดคำสั่งซอฟต์แวร์ เป็นระยะเวลา 30 วัน นับจากวันส่งมอบงานงวดสุดท้าย โดยความรับประกันนี้ไม่ครอบคลุมถึงความเสียหายที่เกิดจากการแก้ไข Source Code โดยบุคคลอื่นที่มิใช่ผู้รับจ้าง หรือการเพิ่มฟีเจอร์การใช้งานใหม่ๆ หลังการส่งมอบ [cite: 42, 43]

### ข้อ 6. การรักษาความลับ
[cite_start]ทั้งสองฝ่ายตกลงที่จะรักษาข้อมูลทางธุรกิจ ข้อมูลทางการค้า ข้อมูลส่วนบุคคลของลูกค้า ตลอดจนเทคนิค วิธีการ หรือข้อมูลใดๆ ที่ได้รับทราบจากการปฏิบัติตามสัญญานี้ไว้เป็นความลับอย่างเคร่งครัด และจะไม่นำไปเปิดเผยต่อบุคคลภายนอก เว้นแต่จะได้รับความยินยอมเป็นลายลักษณ์อักษรจากอีกฝ่ายหนึ่งก่อน [cite: 44, 45]

### ข้อ 7. การบอกเลิกสัญญา
[cite_start]หากคู่สัญญาฝ่ายใดฝ่ายหนึ่งละเลยไม่ปฏิบัติตามเงื่อนไขแห่งสัญญานี้ในข้อหนึ่งข้อใด และไม่ดำเนินการแก้ไขให้ถูกต้องภายในระยะเวลา 15 วัน นับจากวันที่ได้รับหนังสือแจ้งเตือนจากคู่สัญญาอีกฝ่าย คู่สัญญาฝ่ายที่มิได้ผิดสัญญามีสิทธิบอกเลิกสัญญาฉบับนี้ได้ทันที และมีสิทธิเรียกร้องค่าเสียหายตามกฎหมาย [cite: 46, 47]

---

[cite_start]สัญญาฉบับนี้ทำขึ้นเป็นสองฉบับ มีข้อความถูกต้องตรงกัน คู่สัญญาได้อ่านและเข้าใจข้อความโดยตลอดแล้ว เห็นว่าตรงตามเจตนาทุกประการ จึงได้ลงลายมือชื่อไว้เป็นสำคัญต่อหน้าพยาน และยึดถือไว้ฝ่ายละหนึ่งฉบับ [cite: 48]
